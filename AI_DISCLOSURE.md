# 2. Purpose and Scope of Use

AI tools were utilized as technical assistants throughout the development process of InfoFlow Simulator. The primary purpose was to accelerate implementation, debugging, documentation, and technical exploration while keeping all major design decisions and research directions human-driven.

Specific applications included:

- **Code development assistance:**  
  AI tools were used to provide reference implementations and boilerplate structures for Python development, including Streamlit dashboard components, pandas-based data processing, SQL query construction, and machine learning pipelines.

- **Debugging and troubleshooting:**  
  AI assistance was used to interpret error messages, diagnose dependency issues, resolve environment configuration problems, and troubleshoot Git workflow issues. This included debugging Python runtime errors, package installation problems, and repository synchronization conflicts.

- **Data processing and modeling support:**  
  AI tools assisted with syntax references and implementation patterns for data cleaning, CSV processing, regression modeling, feature engineering, and evaluation metric calculation.

- **Documentation and communication support:**  
  AI was used to improve document structure, standardize technical descriptions, and assist in formatting project documentation such as README files, proposals, and reports.

- **Research exploration:**  
  AI tools were used to help identify relevant theoretical concepts, organize literature review directions, and improve explanations of technical contributions.

---

# 3. Methodology of Use

AI-generated suggestions were incorporated through a human-in-the-loop workflow.

The general process was:

1. Identify a specific technical or documentation problem.
2. Consult AI tools for possible solutions, references, or implementation examples.
3. Review and adapt the generated suggestions according to project requirements.
4. Manually test, validate, and modify all adopted outputs before integration.

All AI-assisted code was reviewed by team members and tested within the project environment before being committed to the repository.

AI-generated suggestions were treated as references rather than final solutions. The team remained responsible for verifying correctness, maintaining code quality, and ensuring consistency between the simulation engine, dashboard, analytical pipeline, and final report.

---

# 4. Independent Contributions and Originality Statement

Although AI tools assisted with implementation efficiency, the core intellectual contributions of this project were developed independently by the team.

The following components were designed and implemented through human decision-making:

- The overall system architecture of the schema-adaptive telemetry framework.
- The simulation logic, including user behavior generation, difficulty progression, dropout mechanisms, and parameter design.
- The definition of behavioral metrics and analytical objectives.
- The schema inference strategy and visualization selection logic.
- The experimental design, stress-testing methodology, and validation criteria.
- The theoretical framing connecting information complexity, cognitive load, and user engagement.

AI tools did not generate the fundamental research questions, experimental design, or core algorithmic decisions.

All final code, analysis results, interpretations, and written content were reviewed and approved by the team.

---

# 5. Data Privacy and Security

No private, confidential, or personally identifiable information (PII) was provided to external AI systems.

All datasets used in this project were synthetically generated behavioral logs created specifically for simulation and testing purposes. No real user data was collected or processed during development.
